# Glossary

> **Purpose**: Comprehensive glossary of terms, concepts, and technologies used in NeoTool.

## A

### Access Token
A short-lived JWT token (default: 15 minutes) used for API authentication. Stateless and contains user information in the token payload.

### ADR (Architecture Decision Record)
A document that captures an important architectural decision made along with its context and consequences.

### Apollo Federation
A GraphQL architecture pattern that allows multiple GraphQL services to compose a single unified schema.

## B

### Backend for Frontend (BFF)
A pattern where a separate backend service is created for each frontend application. NeoTool uses GraphQL Federation instead.

## C

### Client Component
In Next.js App Router, a React component that runs on the client side and can use browser APIs and interactivity. Marked with `'use client'` directive.

## D

### Domain-Driven Design (DDD)
An approach to software development that centers the development on programming a domain model that has a rich understanding of the business domain.

## G

### GraphQL
A query language for APIs and a runtime for executing those queries.

## J

### JWT (JSON Web Token)
A compact, URL-safe means of representing claims to be transferred between two parties.

## M

### Micronaut
A modern, JVM-based, full-stack framework for building modular, easily testable microservice and serverless applications.

## S

### Server Component
In Next.js App Router, a React component that runs on the server and can access server-side resources directly.
