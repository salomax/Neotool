# Key Concepts

> **Purpose**: Explanation of key concepts used in NeoTool.

## Spec-Driven Development

Development approach where the specification drives implementation, ensuring consistency and quality.

## GraphQL Federation

Distributed GraphQL architecture pattern enabling multiple services to compose a unified schema.

## Clean Architecture

Layered architecture with clear separation of concerns: API → Service → Repository → Entity.
