# Code Review Workflow

> **Purpose**: Code review process and guidelines.

## Process

1. **Create PR**: Use PR checklist
2. **Review**: Use code review checklist
3. **Address Feedback**: Make requested changes
4. **Approve**: Once all checks pass
5. **Merge**: Merge to main branch

## Guidelines

- Be constructive and respectful
- Focus on code quality and spec compliance
- Use code review checklist
- Request changes for spec violations
